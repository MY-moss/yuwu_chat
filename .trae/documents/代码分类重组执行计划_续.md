## 二、分区注释标准（沿用前序方案）

采用「文件头 + 区块头 + 文件尾」三级结构：

**Python（app.py）**
```python
# ============================================================
# 文件: app.py | 职责: Flask主应用 | 区块数: 22
# ============================================================

# ============================================================
# 区块 22 · 启动入口
# ============================================================
```

**JavaScript（script.js / feedback.js）**
```javascript
// ============================================================
// 文件: script.js | 职责: 前端主逻辑 | 区块数: 18
// ============================================================

// ============================================================
// 区块 03 · 认证UI
// ============================================================
```

**CSS（style.css / feedback.css）**
```css
/* ============================================================
   文件: style.css | 职责: 主样式表 | 区块数: 9
   ============================================================ */

/* ============================================================
   区块 04 · RPG 模式样式
   ============================================================ */
```

**规则**：
1. 每文件首行加文件头
2. 区块头用 60 个 `=` 包裹
3. 区块编号跨文件统一两位数字
4. 文件末尾加 `# ===== END OF FILE =====`（或 `// ===== END OF FILE =====`、`/* ===== END OF FILE ===== */`）

## Assumptions & Decisions

1. **保持单文件架构**：不物理拆分文件，仅通过分区注释实现逻辑分类
2. **纯注释变更**：所有代码修改仅限注释和空行，不改动任何运行时逻辑
3. **release/app.py 删除安全**：PyInstaller 打包后 .exe 自包含，不依赖源码
4. **启动.bat 优先 .exe**：检测到 release/tavern.exe 则启动发布版，否则回退到 python 开发版
5. **版本号 v2.1.5**：本次为代码结构优化，minor 版本+1
6. **区块编号跨文件独立**：每个文件从 01 开始独立编号，不强求跨文件对齐
