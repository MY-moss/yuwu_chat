# docs 文档冗余维修与 AI 工作流指南计划

## Context

`docs/` 目录现有 8 个文件，存在三类问题：
1. **严重冗余**：`项目记忆.md` 与 `项目架构和相关规范.md` 有 5 个章节重叠（60-90% 重复），仅"待办 Top5"和"风险 Top3"是独有内容。
2. **过时文件**：`优化建议.md` 中 11 项已执行项与 `已修复历史问题.md` 重复记录，20 项待审批项中约 8 项与 `问题反馈与报告.md` 交叉，且存在编号错误（两个"七"节）。`AI行为评估报告.md` 是评估脚本的自动生成产物，不应版本控制。
3. **缺少工作流入口**：用户要求"形成一个完整的工作流统计文档，每一次 AI 调用时告诉 AI 如何使用这些文本"。

**目标**：精简 docs 架构至 5 份核心文档 + 1 份工作流指南，消除冗余，建立权威来源机制，确保 AI 每次调用都有明确的文档使用指引。

## 二、AI 调用标准流程
### 2.1 接收任务时（必做）
1. 读取本文件了解文档体系
2. 读取「项目架构和相关规范.md」了解架构和规则
3. 读取「问题反馈与报告.md」了解现存问题
4. 若涉及代码修改，读取「代码分类索引.md」定位区块
5. 运行 `python docs/AI行为评估脚本.py --auto` 基线评估

### 2.2 执行任务时（必遵守）
- 唯一修改入口：src/backend/
- 同步规则：修改后同步到 release/
- 分区注释：新增功能时确定所属区块
- 优化建议：需用户审批后才能执行

### 2.3 完成任务时（必检查）
1. 代码验证：python 语法 + 区块数 + END 标记
2. 文档更新检查清单（按顺序）：
   - [ ] 问题反馈与报告.md（若修复了问题）
   - [ ] 已修复历史问题.md（若修复了问题）
   - [ ] 项目架构和相关规范.md（若版本/架构变更）
   - [ ] 代码分类索引.md（若新增/删除区块）
   - [ ] 优化建议.md（若执行了优化项）
3. 运行 `python docs/AI行为评估脚本.py --auto` 终态评估
4. Git 提交推送

- 版本号规则：架构变更 minor+1，问题修复 patch+1

## 三、权威来源机制

| 信息类型 | 权威文件 | 其他文件做法 |
|---------|---------|------------|
| 项目定位/技术栈/端口/版本 | 项目架构和相关规范.md | 引用，不重复 |
| 目录结构/关键路径 | 项目架构和相关规范.md | 引用 |
| 修改规则/AI接手规则 | 项目架构和相关规范.md + AI工作流指南.md | 引用 |
| 版本历史/变更记录 | 项目架构和相关规范.md | 引用 |
| 现存问题清单 | 问题反馈与报告.md | 引用数字，不展开 |
| 已修复问题档案 | 已修复历史问题.md | 引用累计数 |
| 代码区块结构 | 代码分类索引.md | 引用区块数 |
| AI 行为评估标准 | AI行为评估脚本.py | 引用 |
| 待办优先级/已知风险 | 项目架构和相关规范.md（七、） | — |
| 待审批优化建议 | 优化建议.md | — |
| AI 工作流/文档使用 | AI工作流指南.md | — |
