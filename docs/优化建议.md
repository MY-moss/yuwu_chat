## 一、代码结构优化

### O2. 统一事件绑定方式（feedback.js）⏳ 待审批
- **问题**: feedback.js 同时使用 `onclick="..."` 模板字符串和 `addEventListener`
- **建议**: 全部改为 `addEventListener`

### O3. 统一 API 调用方式（script.js）⏳ 待审批
- **问题**: script.js 中 11 次 `fetch()` 但仅部分使用 `api()` 辅助函数
- **建议**: 全部替换为 `api()` helper

### O4. 消除全局 escapeHtml 重复定义 ⏳ 待审批
- **问题**: escapeHtml 在 script.js、feedback.js、dashboard.html、spectate.html 各自定义，实现不一（feedback.js 不转义 `"`）
- **建议**: 提取为 `static/utils.js` 公共模块，统一实现并转义 `"`，所有文件引用

## 二、性能优化

### O5. JSON 文件缓存层 ⏳ 待审批
- **问题**: `load_agents()`、`load_worlds()` 等每次请求都读取磁盘
- **建议**: 添加带 TTL 的内存缓存层

### O7. 添加数据库索引 ⏳ 待审批
- **问题**: `Feedback.user_id`、`User.last_active` 缺索引
- **建议**:
  ```python
  class Feedback(db.Model):
      user_id = db.Column(db.Integer, db.ForeignKey('user.id'), index=True)
  class User(db.Model):
      last_active = db.Column(db.DateTime, index=True)
  ```

### O8. 优化 feedback_stats 性能 ⏳ 待审批
- **问题**: `ratings.count(i)` 对每个 i 全表扫描，O(5n)
- **建议**: 改用 `collections.Counter` 一次遍历

### O9. generate_credit_key 批量插入 ⏳ 待审批
- **问题**: 循环内 `db.session.add(ck)` 100 次
- **建议**: 改用 `db.session.bulk_save_objects(keys)`

## 三、代码质量优化

### O11. 为 `rpg_sessions` 添加 TTL 清理 ⏳ 待审批
- **问题**: 跑团会话无超时清理，内存持续增长
- **建议**: 在 `save_sessions()` 中清理超过 24 小时未活动的会话

### O12. `int()/float()` 转换异常保护 ⏳ 待审批
- **问题**: 多处直接调用未捕获 `ValueError`
- **建议**: 使用 `safe_int`/`safe_float` 辅助函数

### O13. JSON 文件原子写入 ⏳ 待审批
- **问题**: 所有 JSON 写入 `open(w)` 直接覆盖，非原子
- **建议**: 改为 temp file + os.rename 模式：
  ```python
  def atomic_write(path, data):
      tmp = path + '.tmp'
      with open(tmp, 'w', encoding='utf-8') as f:
          json.dump(data, f, ensure_ascii=False, indent=2)
      os.replace(tmp, path)
  ```

### O14. session_id 增强唯一性 ⏳ 待审批
- **问题**: `str(uuid.uuid4())[:8]` 仅 32 位熵，~65K 会话即 50% 碰撞
- **建议**: 改为完整 uuid4 或加入碰撞检测重试

### O23. 密钥字段脱敏返回 ⏳ 待审批
- **问题**: H1/H22/H23 多处密钥明文返回
- **建议**: 所有密钥字段统一返回 `"***...***"` 脱敏，仅写入时接受明文

## 四、用户体验优化

### O15. Toast 时长可配置 ⏳ 待审批
- **建议**: `function toast(msg, type, duration = 2500)`

### O16. 前端 `catch {}` 添加用户反馈 ⏳ 待审批
- **问题**: script.js 约14处 catch 静默吞异常
- **建议**: 添加 `toast(msg, 'error')` 或 `console.warn()`

### O17. 改善 loading 蒙层处理 ⏳ 待审批
- **问题**: jumpToRound/startGame/resumeGame 无 try/catch，网络错误时蒙层卡死
- **建议**: 用 try/finally 包裹，确保 hideLoading 总执行

### O18. 统一 loading 蒙层逻辑（actGame）⏳ 待审批
- **问题**: `actGame:811-819` 逻辑反转，蒙层可能永久卡住
- **建议**: 修正条件判断，确保所有路径都 hideLoading

## 五、其他优化

### O26. 添加 ARIA 角色与属性 ⏳ 待审批
- **问题**: feedback.html modal、spectate.html 自动刷新区域缺 ARIA
- **建议**: dialog 加 `role="dialog" aria-modal="true"`；spectate 加 `aria-live="polite"`

### O27. label 关联 input ⏳ 待审批
- **问题**: index.html、feedback.html 多处 label 未关联
- **建议**: 添加 `for`/`id` 关联

### O28. 添加 `prefers-reduced-motion` 媒体查询 ⏳ 待审批
- **问题**: style.css 大量动画无禁用选项
- **建议**: `@media (prefers-reduced-motion: reduce) { * { animation: none; } }`

### O30. marked 锁定版本 ⏳ 待审批
- **问题**: `marked.min.js` 未锁定版本（无 `@版本号`）
- **建议**: 改为 `marked@12.0.0`
