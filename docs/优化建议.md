## 二、性能优化

### O5. JSON 文件缓存层 ⏳ 待审批
- **问题**: `load_agents()`、`load_worlds()` 等每次请求都读取磁盘
- **建议**: 添加带 TTL 的内存缓存层

### O6. 消除重复 SQL 查询 ✅ 已执行
- **位置**: app.py:1206-1211 `admin_stats()`
- **变更**: 合并为一次查询 + Python `len()`

### O7. 添加数据库索引 ⏳ 待审批
- **问题**: `Feedback.user_id`、`User.last_active` 缺索引
- **建议**:
  ```python
  class Feedback(db.Model):
      user_id = db.Column(db.Integer, db.ForeignKey('user.id'), index=True)
  class User(db.Model):
      last_active = db.Column(db.DateTime, index=True)
  ```

### O8. 优化 feedback_stats 性能 ⏳ 待审批
- **问题**: `ratings.count(i)` 对每个 i 全表扫描，O(5n)
- **建议**: 改用 `collections.Counter` 一次遍历

### O9. generate_credit_key 批量插入 ⏳ 待审批
- **问题**: 循环内 `db.session.add(ck)` 100 次
- **建议**: 改用 `db.session.bulk_save_objects(keys)`

## 三、代码质量优化

### O10. 为 `histories` 字典添加上限 ✅ 部分执行（修复异常保护，未加上限）
- **问题**: histories 字典无限增长
- **建议**: 每键保留最近 50 条；总键数上限 1000；加 `_histories_lock` 保护

### O11. 为 `rpg_sessions` 添加 TTL 清理 ⏳ 待审批
- **问题**: 跑团会话无超时清理，内存持续增长
- **建议**: 在 `save_sessions()` 中清理超过 24 小时未活动的会话

### O12. `int()/float()` 转换异常保护 ⏳ 待审批
- **问题**: 多处直接调用未捕获 `ValueError`
- **建议**: 使用 `safe_int`/`safe_float` 辅助函数

### O13. JSON 文件原子写入 ⏳ 待审批
- **问题**: 所有 JSON 写入 `open(w)` 直接覆盖，非原子
- **建议**: 改为 temp file + os.rename 模式：
  ```python
  def atomic_write(path, data):
      tmp = path + '.tmp'
      with open(tmp, 'w', encoding='utf-8') as f:
          json.dump(data, f, ensure_ascii=False, indent=2)
      os.replace(tmp, path)
  ```

### O14. session_id 增强唯一性 ⏳ 待审批
- **问题**: `str(uuid.uuid4())[:8]` 仅 32 位熵，~65K 会话即 50% 碰撞
- **建议**: 改为完整 uuid4 或加入碰撞检测重试

## 五、安全加固

### O19. 添加安全响应头 ✅ 已执行
- **变更**: 在 `@app.after_request` 中添加 X-Content-Type-Options、X-Frame-Options、X-XSS-Protection

### O20. 启动信息脱敏 ✅ 已执行
- **变更**: 改为仅打印 API 是否已配置

### O21. SSRF 防护 ✅ 已执行
- **问题**: `user_api_config` PUT 不校验 api_base，可访问内网
- **变更**: 添加 `is_safe_url()` 函数校验：拒绝 localhost/127.0.0.1/0.0.0.0/::1、10.x/172.16-31.x/192.168.x、169.254.x 私有地址

### O22. redeem_key 原子化 ✅ 已执行
- **问题**: redeem_key TOCTOU 竞态可双花
- **变更**: 使用 `CreditKey.query.filter_by(id=key.id, used=False).update({...})` 原子更新，检查返回值确保只有一个事务成功

### O23. 密钥字段脱敏返回 ⏳ 待审批
- **问题**: H1/H22/H23 多处密钥明文返回
- **建议**: 所有密钥字段统一返回 `"***...***"` 脱敏，仅写入时接受明文

### O24. list_shared_sessions 加认证 ⏳ 待审批
- **问题**: 无需登录可枚举所有分享对局
- **建议**: 添加 `@login_required`

### O25. 上游 API 错误不回传 ⏳ 待审批
- **问题**: chat/call_ai 把上游错误正文回传用户
- **建议**: 仅记录日志，返回通用错误信息
