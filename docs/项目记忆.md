## 六、AI 接手规则

### 6.0 执行前必读文档（与项目架构和相关规范.md同等地位）
1. **项目架构和相关规范.md** — 架构规范、修改规则
2. **AI行为评估脚本.py** — 评估标准，规范AI行为
3. **项目记忆.md** — 项目快照，待办优先级
4. **问题反馈与报告.md** — 现存问题清单

### 6.1 开始工作前必做
1. 阅读本文件 — 了解项目状态和待办事项
2. 阅读「项目架构和相关规范.md」— 理解目录结构和修改规则
3. 阅读「问题反馈与报告.md」— 了解当前待修复问题（127项，Critical归零）
4. 阅读「优化建议.md」— 如需执行优化项，先逐项向用户审批

### 6.2 修改代码时必遵守
1. **唯一修改入口** — 所有代码修改必须在 `src/backend/` 目录进行
2. **同步规则** — 修改完成后同步到 `release/`（前端文件及数据文件）
3. **禁止操作** — 直接在 `release/` 或 `云雾酒馆_发布版/` 中修改代码

### 6.3 修改完成后必检查
1. **版本同步** — `version.json` 和 `CHANGELOG.json` 是否一致
2. **安全检查** — 新增的 API 路由是否有正确的认证装饰器
3. **文档更新** — 按顺序更新三个文档
