## 三、修改规则（强制执行）

### 3.1 修改入口
- **唯一入口**: 所有代码修改必须在 `src/backend/` 目录进行
- **同步规则**: 修改完成后同步到 `release/`（前端文件及数据文件）
- **禁止操作**: 直接在 `release/` 或 `云雾酒馆_发布版/` 中修改代码

### 3.2 关键约束
- 端口必须为 9000
- 仅使用 SiliconFlow API
- 管理员账户: admin/123456
- JSON_AS_ASCII = False
- JSON 文件操作必须使用 threading.Lock() 保护

### 3.3 修改后检查清单
- [ ] `version.json` 和 `CHANGELOG.json` 在 `src/backend/`、`release/` 间是否一致
- [ ] 新增的 API 路由是否有正确的认证装饰器
- [ ] 新增的 API 返回值中是否包含 `api_key` 等敏感字段（应过滤）
- [ ] `build.bat` 的路径引用是否指向 `src/backend/`
- [ ] 所有 `start.ps1` 是否都包含 `Set-Location $PSScriptRoot`
- [ ] 文档更新（问题反馈与报告.md → 项目架构和相关规范.md → 项目记忆.md）
- [ ] **AI行为评估脚本.py** 执行前后必读，任务完成后运行自评

### 3.4 AI 行为评估脚本使用规范
- **执行前必读**: AI 在开始任何任务前，必须阅读 `docs/AI行为评估脚本.py` 了解评估标准
- **执行后自评**: AI 完成任务后，必须运行 `python docs/AI行为评估脚本.py --auto` 进行自评
- **反馈处理**: 根据评估反馈改进不足之处，评分低于70分必须重新审视工作
- **报告保存**: 评估报告自动保存到 `docs/AI行为评估报告.md`
