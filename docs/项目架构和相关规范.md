## 三、修改规则（强制执行）

- **唯一入口**: 所有代码修改必须在 `src/backend/` 目录进行

- **禁止操作**: 直接在 `release/` 或 `云雾酒馆_发布版/` 中修改代码

- JSON_AS_ASCII = False

### 3.3 修改后检查清单

- [ ] **AI行为评估脚本.py** 执行前后必读，任务完成后运行自评

### 3.4 AI 行为评估脚本使用规范
- **不可修改**: 不可对`docs/AI行为评估脚本.py` 进行规则上的修改，除非是代码有问题或者技术有问题,或者已经通过管理员的审核后批准，或者管理员要求这么做
- **执行前必读**: AI 在开始任何任务前，必须阅读 `docs/AI行为评估脚本.py` 了解评估标准
- **执行后自评**: AI 完成任务后，必须运行 `python docs/AI行为评估脚本.py --auto` 进行自评
- **反馈处理**: 根据评估反馈改进不足之处，评分低于70分必须重新审视工作
- **报告保存**: 评估报告自动保存到 `docs/AI行为评估报告.md`
每一轮要求重新审核，之前的分数不影响后面的分数，每一次审核独立，并且有历史记录

### 3.5 代码上传规范
- **及时上传**: 所有修改必须及时上传到github仓库
- **及时上传**: 所有修改必须及时通过git上传到github仓库，包括新增文件、修改文件、删除文件等

### 3.6 代码分区规范（v2.1.5 新增）
- **分区注释标准**: 所有源码文件采用「文件头 + 区块头 + 文件尾」三级结构，详见 `docs/代码分类索引.md`
- **区块编号规则**: 每文件独立编号，两位数字（01-22），跨文件不强制对齐
- **子区块细分**: 单区块过大时用 `# --- 子区块名 ---`（Python）或 `// --- 子区块名 ---`（JS）细分，不计入区块数
- **END 标记**: 文件末尾必须保留 `# ===== END OF FILE =====` 标记
- **维护规则**: 新增功能时先确定所属区块；新增区块需更新文件头区块数并同步 `docs/代码分类索引.md`
