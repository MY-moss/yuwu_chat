# 代码分类索引

> 本文档为项目代码的逻辑分类导航，配合各源文件的「区块分区注释」使用。
> 所有代码文件保持单文件架构，通过分区注释实现逻辑分类。

---

## 一、分区注释标准

采用「文件头 + 区块头 + 文件尾」三级结构：

### Python（app.py）
```python
# ============================================================
# 文件: app.py | 职责: Flask主应用 | 区块数: 22
# ============================================================

# ============================================================
# 区块 04 · 数据库模型
# ============================================================
```

### JavaScript（script.js / feedback.js）
```javascript
// ============================================================
// 文件: script.js | 职责: 前端主逻辑 | 区块数: 18
// ============================================================

// ============================================================
// 区块 03 · 认证UI
// ============================================================
```

### CSS（style.css / feedback.css）
```css
/* ============================================================
   文件: style.css | 职责: 主样式表 | 区块数: 9
   ============================================================ */

/* ============================================================
   区块 04 · RPG模式
   ============================================================ */
```

### 规则
1. 每文件首行加**文件头**（文件名 + 职责 + 区块数）
2. **区块头**用 60 个 `=` 包裹，编号统一两位数字
3. 子区块用 `# --- 子区块名 ---`（单行，仅作细分，不计入区块数）
4. 文件末尾加 `# ===== END OF FILE =====`（或对应注释语法）

## 七、维护规则

1. **新增功能时**：先确定所属区块，在对应区块末尾添加代码；若需新开区块，更新文件头区块数并同步本文档。
2. **区块编号连续**：不得跳号或重号；删除区块时需重新编号。
3. **子区块细分**：单区块过大时可用 `# --- 子区块名 ---`（Python）或 `// --- 子区块名 ---`（JS）细分，不计入区块数。
4. **同步规则**：修改 `src/backend/` 后必须同步到 `release/` 对应文件。
5. **END 标记**：文件末尾必须保留 `# ===== END OF FILE =====` 标记，便于完整性校验。
6. **纯注释变更**：分区注释为纯注释，不影响运行时行为；可通过 `git diff --stat` 验证仅注释行变动。
